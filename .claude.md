## Project Overview

ENCT v1.3 Agent Product Development using Harness Engineering methodology.

**What we're building:** An agent with Enactive Normative Control Theory framework
**How we're building it:** Using Harness Engineering to ensure quality and safety

---

## File Organization Rules

### Documentation
- `/enct/` — ENCT theory, specs, governance (the product specification)
- `/harness/` — HE audit, gaps, plans (the development methodology)
- Root — Project-level (ENCT-TASK-HIERARCHY.md, RULES.md)

### Code
- `/src/enct/` — ENCT engine (shipped)
- `/src/harness/` — HE tooling (development-only, NOT shipped)
- `/src/api/`, `/src/ui/`, `/src/storage/`, `/src/monitoring/` — Supporting services
- `/src/tests/` — Test suites

---

## Critical Separation

**ENCT and Harness Engineering are ORTHOGONAL:**
- ENCT is the target (agent product)
- HE is the methodology (how to build safely)
- They do NOT integrate at runtime
- HE is discarded after deployment

See `/enct_vs_harness_clarification.md` in memory for full details.

---

## When Writing Code

1. **ENCT Implementation:** Code goes in `/src/enct/[component]/`
2. **Harness Tooling:** Code goes in `/src/harness/[component]/` (dev-time only)
3. **Tests:** Code goes in `/src/tests/`
4. **Never:** Put code at root or mix ENCT/HE imports

---

## When Writing Docs

1. **ENCT Specs:** Docs go in `/enct/` folder
2. **HE Guidance:** Docs go in `/harness/` folder
3. **Project-Level:** Docs go in root (RULES.md, ENCT-TASK-HIERARCHY.md)

## Task Tracking (Tier 2)

Use TaskCreate/TaskUpdate for Phase 2+ work:

1. **Create tasks** for Phase 2 work (see ENCT-TASK-HIERARCHY.md)
   - Format: `TaskCreate` with subject + description
   - Example: "Implement 5-phase loop validation logic"
   
2. **Link to hierarchy:** Tasks correspond to rows in ENCT-TASK-HIERARCHY.md
   - Each task has dependencies (blockedBy, addBlocks)
   - Run tasks in dependency order (critical path first)

3. **Update on progress:**
   - Set status: pending → in_progress → completed
   - Add notes if blocked
   - Mark completed when done

4. **Phase Structure:**
   - Phase 2 (Training): Implement ENCT engine + harness
   - Phase 3 (Testing): Run verification + tests
   - Phase 4 (Deploy): Package for users
   - Phase 5 (Monitor): Production observability
